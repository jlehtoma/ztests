Zonation 3.1.9 benchmarks

Main questions:

  1. How fast does Zonation run on various hardware resources?
  2. How hast does Zonation run on various operating systems?

All code for running these tests and for producing this document can be found here. Data cannot be shared publically.

Previous benchmark with Zonation tutorial material can be found here.

TL;DR:

  • A static build of Zonation performs ~1.8 times faster on Linux than on Windows running on a machine with new-generation CPU (i.e. on the same computer).
  • Zonation runinnig on a Windows machine with new-generation CPU performs ~1.4 times faster than a Windows machine with old-generation CPU (i.e. on different computers).
  • A static build of Zonation performs 2.6 times faster on Linux machine with new-generation CPU than on Windows running on a machine with old-generation CPU (i.e. on different computers).

Zonation version

Version 3.1.9 was used on all machines. Linux version is a static build whereas Windows version is the one installed by the installer (dynamically linked).

Data

Data from real-life analysis from the forest center of South-Savo is used. 20 forest structure index layers form the basis of the biodiversity features. This stack is duplicated accordingly if the analysis setup (e.g. matrix connectivity) requires so.

While the analysis variants used (N=6) in this benchmark are not huge (~4 million effective cells), they are big enough to give an idea on Zonation performance in real-life planning related analysis.

Benchmark

All analysis runs were run 1 at a time. All time measures are parsed from the resulting run-info file, so they are the ones reported by Zonation.

Out of these comparisons, the Linux machine performs the fastest. By scaling MRGTESLA to 1.0, the others perform in a following way:

Averaging over all the variants, a Linux machine performs ~1.77 times faster.

Looking at the time needed for initializing the analysis (reading in the data + doing connectivity smoothings etc) show some differences as well.

Better performance on LH2-BIOTI25 may be explained by the SSD disk.

Comparing rank rasters

One essential question is, whether Zonation produces the same (or similar enough) results on different platforms. Following table summarizes the differences between different variants produced on two systems: LH-BIOTI25 (Win) and LH-BIOTI25 (Linux). Table columns mean the following:

Overall similarity

Jaccard index is a statistic used for comparing the similarity and diversity of sample sets.

jaccard.threshold: which proportion of the ranks is compared using Jaccard index. Complement to “top-fraction”, i.e. 0.99 means the best 1% of the landscape.

jaccard.index: value of the Jaccard index. Value 1 means that the compared sets overlap perfectly.

Kendall tau rank correlation coefficient is a measure of rank correlation, i.e., the similarity of the orderings of the data when ranked by each of the quantities.

kendall.tau: The tau statistic.

kendall.tau.p: The two-sided p-value for a hypothesis test whose null hypothesis is an absence of association, tau = 0.

Cell-wise differences

Rest of the columns are statistics for cell-wise differences produced by

rank_raster1 - rank_raster2

max: largest positive (raster1 > raster2) difference in rank value between two cells
mean: mean difference in rank value between two cells
min: largest negative (raster1 < raster2) difference in rank value between two cells
std: standard deviation of all differences

Observations:

  • Zonation running on both operating systems produces exactly the same results for variants 15 and 16.
  • Variants 17, 18, 19, and 20 have slightly different, but very similar results.
  • The complexity of the variants is 15 < 16 < 17 < 18 < 19 < 20. More specically, connectivity matrix is introduced in variant 17 and varying number of interactions in 18 and onwards.

From Fede:

“Results (rank and curves) should be definitely the same (in a soft sense, with small differences due to randomization/ties) and floating point precision.”

This probably explains the differences in results (I did not look at the curve files) as the differences get larger as new connectivity measures are introduced.
